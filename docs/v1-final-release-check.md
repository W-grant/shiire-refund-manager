# Version1 最終リリース判定チェック

作成日: 2026-07-08

Version1を本番運用開始してよいかを判断するための最終チェックシート。

## 現在の判定

コード、ドキュメント、自動テスト、Cloudflare Production反映はリリース直前水準。

残る判定は、実務データと実ファイルを人が確認する工程。

## 自動確認済み

- 構文チェック
- 分類ロジックテスト
- CSV読み書きテスト
- Cloudflare / Netlify Functions互換テスト
- ZIP出力テスト
- UI文言退行防止テスト
- 販売管理・経営ダッシュボード・Google Sheets重要UI退行防止テスト
- USER_GUIDEの販売管理・ダッシュボード・Google Sheets説明チェック
- Vite production build
- GitHub main push
- Cloudflare Production最新commit反映
- Production URL表示

## 人の確認が必要

### 1. admin実務データ確認

- adminでログインできる
- 実際の仕入を登録できる
- 複数証憑画像を添付できる
- 一覧に代表サムネイルが表示される
- 証憑詳細で画像を切り替えられる
- 登録済み仕入を編集できる
- 仕入を削除できる

### 2. AI抽出確認

- `/extract` のヘルスチェックで `anthropicConfigured: true`
- 画像または明細テキストを読み取れる
- 抽出候補を目視修正できる
- 修正後に仕入登録できる

### 3. 月次提出確認

- 対象月を選択できる
- 税理士提出ZIPを作成できる
- 保存履歴に表示される
- 保存履歴から再ダウンロードできる
- 管理者で保存履歴を削除できる

### 4. 販売管理・利益確認

- 仕入レコードに販売情報を登録できる
- 販売先を CATAWIKI / EBAY / その他から選べる
- 売却済みでは販売日と販売価格が必須になる
- 販売価格、手数料、送料から粗利が表示される
- 販売先、状態、担当、商品名・管理番号・SKUで絞り込める
- 経営ダッシュボードで対象月を切り替えられる
- 月別売上、月別粗利、利益率を確認できる
- 担当者別粗利と販売先別粗利を確認できる
- 未処理タスクの内訳を確認できる

### 5. Googleスプレッドシート連携確認

- `スプシへ送信` が成功する
- `仕入管理` に仕入データが反映される
- `販売管理` に販売データが反映される
- `CATAWIKI` タブが作成・更新される
- `EBAY` タブが作成・更新される
- `同期ログ` に同期結果が記録される

### 6. 実ファイル確認

- `01_仕入一覧.csv` をWindowsで開ける
- `02_古物商特例帳簿.csv` をWindowsで開ける
- `03_電帳法索引簿.csv` をWindowsで開ける
- `04_月次サマリー.xlsx` をExcelで開ける
- `05_月次レポート.pdf` をPDFビューアで開ける
- `証憑/` フォルダ内の画像を開ける

### 7. 本番データ整理

- 不要なテスト仕入を削除した
- 不要なテスト販売情報を削除した
- 不要なテスト証憑を削除した
- 不要なテスト提出ZIP履歴を削除した
- Supabase `purchases` に不要な未削除データがない
- Supabase `sales` に不要な未削除データがない
- Supabase Storage `evidence` / `tax-packages` に不要なテストファイルがない

## Release判定

以下を満たせばVersion1 Release可。

- adminが日常入力を完了できる
- AI抽出は補助機能として使える
- 販売管理と経営ダッシュボードで売上・粗利を確認できる
- Googleスプレッドシートへ仕入・販売データを送信できる
- 月次税理士提出ZIPを税理士へ渡せる
- Supabase Database / Storageに保存される
- Production URLが最新commitで配信されている
- 不要なテストデータが残っていない
