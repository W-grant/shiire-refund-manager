## 現在の基本方針

- 当面はadmin中心の少人数運用で開始できる状態を目指す
- staff / tax_accountant は必要になった時点で追加する
- 既に設計と実装は3権限に対応済み
- 本番テストでは不要なテストデータを残さない

## P0: Version1 Release前に必須

| 順位 | 作業 | 目的 | 完了条件 |
| --- | --- | --- | --- |
| 1 | admin実務データテスト | 実際の仕入業務で使えるか確認 | 仕入登録、編集、削除、証憑保存が完了する |
| 2 | AI抽出の実務確認 | 読み取り精度とレビュー運用を確認 | 画像またはテキストから候補を作り、確認後に保存できる |
| 3 | 月次税理士提出ZIP確認 | 月次提出が業務で成立するか確認 | CSV/XLSX/PDF/証憑入りZIPを作成し、税理士側で開ける |

| 5 | 本番データ整理 | テストデータ混入を防ぐ | 不要な仕入、証憑、提出ZIP履歴が残っていない |
| 6 | README / 説明書の最終確認 | 運用担当が迷わないようにする | Cloudflare / Supabase前提の手順に揃っている |

## 次の具体的な進め方

1. adminで実務仕入を数件登録する
2. 証憑画像を複数枚添付する
3. AI抽出からフォーム反映、保存まで確認する
4. 月次税理士提出ZIPを作成する
5. 保存履歴からZIPを再ダウンロードする
6. 税理士または社内担当にファイルを開いてもらう
7. 指摘があればUI文言、帳票、出力形式を修正する
8. Version1 Release判定を行う

## 合格条件

- Production URLで最新commitが配信されている
- adminが本番で一連の業務を完了できる
- Supabase Database / Storageに保存される
- AI抽出がCloudflare `/extract` 経由で動く
- 税理士提出ZIPが作成・保存・再DL・削除できる
- READMEとUSER_GUIDEが現在のCloudflare / Supabase構成と一致している
