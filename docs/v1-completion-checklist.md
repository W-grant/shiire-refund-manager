# Version1 完成チェックリスト

作成日: 2026-07-01

この文書は Version1 完成に向けた現状棚卸しです。ここでは実装変更は行わず、現在の到達点、未完成部分、不安定な部分、Supabase 化に入る前に直すべき点を整理します。

## 現在の全体状況

現在のアプリは、仕入登録、古物商特例判定、複数証憑画像、CSV/Excel/PDF 出力、月次の税理士提出パッケージ ZIP まで実装済みです。

一方で、保存基盤はまだ IndexedDB を中心としたローカル保存です。Netlify Blobs を使う同期機能はありますが、Version1 完成条件に含まれている Supabase Database / Storage 保存は未実装です。

そのため、現時点の判定は「ローカル運用版としては主要機能が揃っているが、Version1 完成条件には未達」です。

## 未完成・不安定な機能

### Supabase保存

現時点では設計とSQLまで完了していますが、アプリ本体は Supabase に接続していません。Version1 完成条件に Supabase保存が含まれているため、最大の未完了項目です。

必要な残作業:

- Supabase Auth ログイン
- 管理者、スタッフ、税理士閲覧用の権限制御
- 仕入データの Supabase Database 保存
- 証憑画像の Supabase Storage 保存
- 月次税理士提出ZIPの Supabase Storage 保存
- IndexedDB 既存データの移行
- RLS の実環境検証

### AI抽出

AI抽出は実装済みですが、実務運用では読み取りミスが起きる前提で使う必要があります。

不安定な点:

- Claude の応答が想定外形式の場合、JSON解析に失敗する可能性がある
- 読み取り精度や信頼度を画面上で確認する仕組みがまだ弱い
- 日付、金額、相手方、匿名/本人確認、適格請求書の判定は人の確認が必須
- 本番の Anthropic API キー、Netlify 環境変数、共有シークレットでの通し確認が必要

### Excel出力

通常のExcel出力は CDN の SheetJS に依存しています。ネットワークやCDN読み込みに失敗した場合、Excel出力が使えません。

税理士提出パッケージ内では最小構成の XLSX 生成代替がありますが、通常Excel出力と完全に統一されていません。

### PDF出力

通常PDFはブラウザ印刷を使っています。税理士提出パッケージ内のPDFは簡易PDF生成です。

不安定な点:

- PDFレイアウト品質が最低限
- 日本語フォント埋め込みに未対応
- 税理士提出用の帳票として見やすい形式に改善余地がある
- PDFファイルの自動検証がない

### 税理士提出パッケージZIP

機能自体は実装済みですが、Version1前に実データでの検証が必要です。

確認したい点:

- ZIP内の日本語ファイル名が Windows / macOS で問題なく開けること
- 同名証憑ファイルの重複回避が期待通り動くこと
- 複数証憑画像がすべて含まれること
- CSV、XLSX、PDFを税理士側で開けること
- 対象月に大量データがある場合もブラウザが固まらないこと

### テスト範囲

現在の自動テストは分類ロジック、CSV、Netlify Functions が中心です。

不足しているテスト:

- 画面操作テスト
- 複数証憑画像の追加、削除、復元テスト
- 税理士提出ZIPの中身検証
- Excel / PDF ファイルの開封検証
- AI抽出レビューから登録までのテスト
- Supabase RLS の権限テスト

## Supabase化に入る前に直すべきバグ・改善点

1. PDF出力の品質改善
   - 税理士提出ZIPに含めるPDFは、日本語が読める状態で安定生成できるようにする。

2. Excel出力の依存整理
   - CDN依存だけにせず、税理士提出ZIPと通常出力で同じ生成方法に寄せる。

3. AI抽出結果の検証強化
   - 金額、日付、数量、相手方が空または異常値の場合に登録前警告を出す。

4. ZIP出力の自動テスト追加
   - 月次ZIPに期待ファイルと全証憑が含まれることを自動確認する。

5. Netlify Blobs 同期の扱い整理
   - Supabase化後に残すのか、移行用・廃止予定とするのかを決める。

6. 既存IndexedDBデータの移行導線
   - バックアップJSONを使った移行か、画面上の移行ボタンにするかを決める。

7. 文字化け・ファイル名検証
   - CSV、ZIP、PDF、Excelで日本語が実務環境のPCで問題なく読めることを確認する。

## Version1 完成条件

Version1 完成と判断するには、少なくとも次の状態まで必要です。

- 管理者、スタッフ、税理士閲覧用の3権限でログインできる
- 仕入データが Supabase Database に保存される
- 証憑画像が Supabase Storage に保存される
- 月次税理士提出ZIPが Supabase Storage に保存される
- IndexedDB の既存データを Supabase に移行できる
- スタッフは登録・編集でき、税理士閲覧用は閲覧・ダウンロードのみできる
- 複数証憑画像、CSV、Excel、PDF、税理士提出ZIPが Supabase 保存後も壊れない
- 3〜5人で同時に社内運用テストを行い、重大な不具合がない
- 本番デプロイ環境の環境変数、RLS、Storage 権限が確認済み

## 現時点の結論

ローカル保存版としての業務機能はかなり揃っています。

ただし、Version1 完成条件に Supabase保存と社内運用テストが含まれているため、現在は Version1 完成前の最終準備段階です。次は Supabase 接続実装に進む前に、出力系の弱点を最低限補強し、その後 Auth / Database / Storage / 移行 / RLS 検証の順で進めるのが安全です。
